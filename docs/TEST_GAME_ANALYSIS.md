# Тестова гра та аналіз системи

Короткий посібник: як запустити тестову гру, переглянути логи та перевірити, що conclusion і trust_snapshot заповнені.

## 1. Запустити тестову гру

З кореня проекту:

```bash
python scripts/run_test_game.py --rounds 3
```

або напряму:

```bash
python run_simulation_live.py --rounds 3 --html --name test_analysis
```

Параметри:

- `--rounds N` — кількість раундів (за замовчуванням 3).
- `--name NAME` — суфікс для файлів логів (за замовчуванням `test_analysis`).
- `--agents id1,id2,...` — обрані агенти; якщо не вказано, беруться з roster.

Після завершення у консолі буде вказано шлях до JSON/HTML логів у `logs/`.

## 2. Відкрити HTML-лог у браузері

У папці `logs/` знайдіть файл `game_YYYY-MM-DD_HH-MM-SS_test_analysis.html` і відкрийте його у браузері. Там буде зведений вигляд гри: раунди, діалоги, очки, переможець.

## 3. Запустити аналіз результатів

```bash
python scripts/analyze_test_game.py test_analysis
```

Скрипт виведе:

- Кількість раундів і переможця з логу.
- Для кожного агента з логу:
  - кількість записів у `game_history`;
  - чи заповнено `conclusion` у останньому записі;
  - наявність і вміст `trust_snapshot`;
  - короткий нарратив з `memory_summary_to_narrative` (перші ~250 символів).

На що звертати увагу:

- **conclusion** — не порожній текст від першої особи (рефлексія після гри).
- **trust_snapshot** — є ключі для інших агентів, значення в межах 0–1.
- **narrative** — українською, без сирого JSON, з фразами на кшталт «Тебе зраджували N разів», «Твій останній висновок: …».

## 4. Перевірити MEMORY.json вручну

Для обраного агента (наприклад `agents/agent_synth_g/`):

1. Відкрийте `agents/<agent_id>/MEMORY.json`.
2. Переконайтеся, що в кінці масиву `game_history` є запис з полями `conclusion` та `trust_snapshot`.
3. Після гри з реальним LLM `conclusion` має бути заповнений; `trust_snapshot` — словник `{ "agent_id": number }`.

Ці кроки дають змогу швидко перевірити, що після тестової гри система коректно зберігає висновки та trust між іграми.
